# Resultados alcançados

## Introdução

Este trabalho, realizado na disciplina de Requisitos de Software (2025.2), teve como propósito identificar, registrar e validar as necessidades das partes interessadas para o desenvolvimento do sistema Loterias Caixa. Ao longo do semestre aplicamos práticas de engenharia de requisitos — desde a elicitação até a verificação dos artefatos — com o objetivo de assegurar que o produto final atendesse às expectativas dos usuários e entregasse valor em cada fase. A seguir, apresentamos de forma mais detalhada os resultados obtidos pelo grupo.

## Resultados Alcançados
 
 No início do projeto, a equipe realizou uma fase de planejamento essencial para alinhar seus membros, definir o escopo, escolher metodologias, criar um cronograma e selecionar as ferramentas de trabalho. Essa etapa inicial foi fundamental para estabelecer uma base sólida e documentar as necessidades do projeto — todo esse conjunto de decisões está registrado em [Planejamento](../Planejamento/).
 
 Para criar requisitos iniciais, fizemos 4 técnicas de elicitação, sendo elas [Análise de Documentos](../Elicitação/Técnicas/Análise%20de%20Documentos.md), [Entrevista](../Elicitação/Técnicas/Entrevista.md), [Observação](../Elicitação/Técnicas/Observação.md) e [Glossário](../Elicitação/Técnicas/Glossário.md). Conseguimos, a partir delas, termos uma noção básica de como elencar requisitos com e sem a participação de usuários. Com essas atividades conseguimos identificar requisitos funcionais e não funcionais tanto com participação direta de usuários quanto a partir de fontes documentais. Em seguida, priorizamos os requisitos com diferentes abordagens, escolhendo a técnica adequada conforme o contexto: [$100](../Elicitação/Priorização/$100.md), [MoSCoW](../Elicitação/Priorização/MoSCoW.md), [Three Level Scale](../Elicitação/Priorização/Three%20Level%20Scale.md) e [Value, Cost and Risk](../Elicitação/Priorização/Value%2C%20Cost%20and%20Risk.md).
 
 Para aprofundar a compreensão sobre os usuários, as funcionalidades do sistema e os requisitos, foram aplicados conceitos práticos de Modelagem, como a criação de [Casos de Uso](../modelagem/Casos%20de%20uso.md), [Cenários](../modelagem/Cenários.md), [Histórias de Usuário](../modelagem/Ágil/História%20de%20usuário.md), [Especificação Suplementar](../modelagem/Especificação%20Suplementar.md), [Léxicos](../modelagem/Léxicos.md), [NFR Framework](../modelagem/Ágil/NFR.md) e [Backlog](../modelagem/Ágil/Backlog.md). Essas técnicas permitiram que a equipe adotasse a perspectiva do usuário, orientando o desenvolvimento para atender às suas expectativas e, ao mesmo tempo, serviram como base para a documentação adequada dos requisitos.
 
 Ao longo do desenvolvimento, a rastreabilidade foi aplicada para conectar os diversos requisitos levantados. Na fase final, implementamos rastreabilidade para relacionar requisitos e entregáveis; [Matriz Geral](../Pós-rastreabilidade/matriz_geral.md) e de artefatos de [Pós-Rastreabilidade](../Pós-rastreabilidade/) consolidaram essas ligações e facilitaram o acompanhamento de mudanças..
 
 Por fim, adotamos verificações sistemáticas dos artefatos — incluindo inspeções no estilo Fagan — para identificar fragilidades e promover correções, contribuindo para a qualidade final dos entregáveis. Todas as inspeções e checklists aplicadas estão documentadas em [Verificação](../Verificação/).
